# AI-DLC 方法论介绍

## 什么是 AI-DLC

AI-DLC（AI-Driven Development Life Cycle）是一套**结构化的 AI 辅助软件开发工作流**，核心思想是：让 AI 不只是写代码，而是全程参与从需求到交付的完整开发生命周期，同时保持**人类在关键决策点的控制权**。

---

## 三个核心阶段

### 🔵 Inception（立项阶段）— 搞清楚"做什么、为什么做"
- 需求分析与验证
- 编写用户故事
- 应用架构设计，拆分并行开发单元
- 风险评估与复杂度评估

### 🟢 Construction（构建阶段）— 搞清楚"怎么做"
- 详细组件设计
- 代码生成与实现
- 构建配置与测试策略
- 质量保证与验证

### 🟡 Operations（运维阶段）— 部署与运行
- 部署自动化与基础设施
- 监控与可观测性
- 生产就绪验证

---

## 关键设计原则

**自适应深度** — 不是每个项目都走完所有阶段。AI 会评估复杂度，简单改动快速处理，复杂变更才走完整流程，避免过度工程化。

**问题驱动** — AI 通过结构化的多选题文件（不是聊天）向你收集信息，答案有据可查、可复现。

**人类审批门控** — 每个阶段结束前，AI 生成执行计划，你审批后才继续。AI 提议，人类决策。

**扩展机制** — 支持叠加自定义规则，比如安全合规、组织内部规范，作为阻断性约束强制执行。

---

## 使用方式

1. 在聊天中以 **"Using AI-DLC, ..."** 开头描述你的需求
2. 工作流自动激活，引导你回答结构化问题
3. 审查 AI 生成的执行计划
4. 逐阶段审批，所有产出物保存在 `aidlc-docs/` 目录

## 支持的 AI 编码工具

AI-DLC 通过在项目目录放置规则文件来工作，支持以下工具：

| 工具 | 规则文件位置 | 规则机制名称 |
|------|------------|------------|
| Kiro | `.kiro/steering/` | Steering Files |
| Amazon Q | `.amazonq/rules/` | Q Rules |
| Cursor | `.cursor/rules/` | Cursor Rules |
| Cline | `.clinerules/` | Cline Rules |
| Claude Code | `CLAUDE.md` | Project Memory |
| GitHub Copilot | `.github/copilot-instructions.md` | Custom Instructions |

> 类比理解：就像给新员工写的《工作手册》——不管谁来干活，都得先读这本手册，按手册里的流程办事。AI-DLC 的 `core-workflow.md` 就是这本手册。

---

## 本质定位

它是一套**方法论，不是工具**。不需要安装任何东西，只需把规则文件放到项目里，任何支持项目级规则的 AI 编码工具都能加载并执行这套工作流。

核心价值：把 AI 的能力约束在一个**可预期、可审计、可重复**的框架里，而不是让它自由发挥。
